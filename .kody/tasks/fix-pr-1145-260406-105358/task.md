# fix: Fix Mixpanel User Identity Merging on Login

## What

The login flow in UserIdentificationTracker.tsx calls alias(user.id) without passing the anonymousId parameter. For proper Mixpanel identity merging, alias() must be called with both the userId and anonymousId (from the mp_anon_id cookie). Currently alias(user.id) is called but without the second parameter, so Mixpanel cannot properly merge the anonymous session with the identified user profile.

- `src/infra/analytics/components/UserIdentificationTracker.tsx`
- `src/infra/analytics/utils/anonymous-id.ts`
- `src/infra/analytics/adapters/mixpanel/adapter.ts`
